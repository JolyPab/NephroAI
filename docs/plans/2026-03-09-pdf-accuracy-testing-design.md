# PDF Parsing Accuracy Testing — Design

**Date:** 2026-03-09
**Goal:** Validate that the V2 PDF extraction pipeline parses medical lab reports with 100% accuracy.

## Context

The extraction pipeline: PDF → OpenAI File API (gpt-5.2) → `ImportV2` structured output → key normalization + scientific unit normalization → DB.

Current tests only cover the deterministic post-processing steps (`_normalize_scientific_units`, `_assign_series_keys`). The LLM extraction step has zero test coverage.

## Approach: LLM-as-Judge

Run extraction on real PDF fixtures, then send the PDF + extracted JSON to a second GPT call that audits correctness and completeness. No manual golden files required.

## Structure

```
backend/tests/accuracy/
├── fixtures/           ← real (anonymized) PDF files
│   ├── sample_01.pdf
│   └── ...
├── judge_prompt.py     ← system prompt for the auditor LLM
├── judge.py            ← async function: (pdf_bytes, extracted_json) → JudgeResult
├── test_accuracy.py    ← pytest, parametrized per PDF in fixtures/
└── report.html         ← generated by pytest-html after each run
```

## Judge Checks

**Critical (test fails):**
- Missing metric — row exists in PDF but absent from JSON
- Wrong numeric value (>1% deviation)
- Wrong unit of measurement
- Wrong reference range boundaries

**Warning (test passes, shown in report):**
- Inaccurate `raw_name`
- Wrong `specimen` classification
- Missing analysis date

## Judge Response Schema

```json
{
  "issues": [
    {
      "severity": "critical | warning",
      "metric": "GLUCOSA",
      "field": "value_numeric",
      "expected": "94.5",
      "got": "94.2",
      "description": "..."
    }
  ],
  "total_metrics_in_pdf": 15,
  "total_metrics_extracted": 15
}
```

## How to Run

```bash
# Place PDFs in backend/tests/accuracy/fixtures/
cd backend && python -m pytest tests/accuracy/ -v --html=tests/accuracy/report.html --self-contained-html

# Single PDF:
cd backend && python -m pytest tests/accuracy/ -v -k "sample_01"
```

## Requirements

- `OPENAI_API_KEY` in environment
- `pytest-html` installed (`pip install pytest-html`)
- ~2 API calls per PDF (extraction + judge)
